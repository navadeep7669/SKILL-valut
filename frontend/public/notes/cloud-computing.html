Understanding Service Models (IaaS, PaaS, SaaS)

Cloud architecture is divided based on what part of the stack the cloud provider manages vs what the developer controls.

ModelStands ForWhat You ManageExamples
IaaSInfrastructure as a ServiceOS, Runtime, Middleware, Code, DBsAWS EC2, Google Compute Engine
PaaSPlatform as a ServiceApplication code and logic configurationsAWS Elastic Beanstalk, Heroku, Vercel
SaaSSoftware as a ServiceNothing (just consume the app)Google Workspace, Microsoft 365, Slack
AWS Core Services Reference
ServiceCategoryPrimary Use CaseBest Practice
EC2ComputeVirtual servers in the cloudSet Auto-Scaling groups with target parameters
S3StorageHighly durable key-value object storageEnable versioning and lifecycle transition rules
VPCNetworkingIsolated private network boundarySeparate public subnets from database private subnets
IAMSecurityAccess management and identity logsEnforce Least Privilege (never use root API keys)
JSON — AWS IAM Least Privilege Policy Example
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject"
      ],
      "Resource": "arn:aws:s3:::skillvault-x-student-data/*"
    }
  ]
}
💡 Rule: Never commit AWS Access Keys or Secret Keys into git files. Always use AWS IAM Roles mounted to EC2 instances instead!

~8 hrs
VMs vs Containers

Virtual Machines virtualize physical hardware, whereas containers share the host operating system kernel, making them extremely lightweight and fast.

AspectVirtual MachinesDocker Containers
IsolationHardware-level (Hypervisor)OS-level (Namespaces & Cgroups)
SpeedMinutes to boot upMilliseconds to boot up
SizeGigabytes (requires full Guest OS)Megabytes (only app + libs)
EfficiencyHigh resource overheadZero hypervisor CPU tax
Dockerfile — Packaging a Node.js App securely
# 1. Use lightweight alpine linux image
FROM node:20-alpine

# 2. Establish isolated working directory
WORKDIR /usr/src/app

# 3. Cache dependencies
COPY package*.json ./
RUN npm ci --only=production

# 4. Copy source files
COPY . .

# 5. Restrict execution privileges (defends against root breakouts)
USER node

# 6. Set production environment & expose port
ENV PORT=3000
EXPOSE 3000

# 7. Start container
CMD ["node", "server.js"]
Kubernetes Architecture Blocks
ObjectPurposeManagement Layer
PodSmallest deployable unit (houses containers)Assigned private IP, shares storage limits
DeploymentManages replica sets and rolling upgradesConfigures scale numbers, handles zero-downtime rollouts
ServiceInternal load balancer directing traffic to podsMaintains stable DNS endpoints for dynamic pods
IngressExternal HTTP/S routing rules routerSSL termination, host-based routing (API gateways)

Infrastructure as Code (IaC) with Terraform

Terraform allows engineers to describe infrastructure state declarative in files. It compares the plan with reality and updates what is necessary.

HCL — Terraform AWS S3 & VPC Configuration
provider "aws" {
  region = "us-east-1"
}

# 1. Create a Secure Private S3 Bucket
resource "aws_s3_bucket" "student_assets" {
  bucket = "skillvault-x-student-data-production"
}

# 2. Block all public access for security
resource "aws_s3_bucket_public_access_block" "block_public" {
  bucket = aws_s3_bucket.student_assets.id

  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}
Standard IaC commands:
  • terraform init: Installs cloud provider plugins
  • terraform plan: Previews updates before making changes
  • terraform apply: Executes plans to build live cloud services
  • terraform destroy: Safely tears down resources to save costs

Phase 4 · Resilient System Design
High Availability, Monitoring & Disaster Recovery
~15 hrs
Designing for 99.99% Availability

Resiliency means designing applications that can survive hardware failures, network splits, or datacentre disasters without losing user data.

PatternMechanicBenefit
Multi-AZDeploying across multiple independent server clustersProtects against local facility failures
Load BalancersDistributing user requests across healthy serversEnsures high-capacity operations and health checking
Read ReplicasOffloading database reads to duplicate nodesPrevents master databases from hitting high load boundaries
Caching (CDN)Caching static assets at global edge locationsDramatically decreases load speed and server traffic
🛡️
Defense
Use rate limiting and API firewalls to block DDoS attacks.
📈
Observables
Set up Prometheus metrics and Grafana alerts for server CPU limits.
🔄
Autoscale
Scale horizontally (add servers) dynamically based on traffic spikes.
💾
Backups
Take regular snapshots and replicate critical databases to other regions.
